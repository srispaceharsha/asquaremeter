Entry 2: Data Integrity

February 03, 2026

Data integrity day. Sri noticed the "Most Frequently Seen" counts were off. We found species name drift—same bug logged as "Carpenter Ant" and "Carpenter Ant (camponotus Parius)". Same with dozens of others. Normalized 383 entries.

Then a subtler bug: two different species sharing the same common name. "Picture-winged Fly" is both Spathulina acroleuca and Physiphora clausa. The logweb UI was deduping them by common name, so one species vanished. Fixed with composite keys.

The pattern: Sri catches data issues before I do. He's living in the data daily, so anomalies surface through intuition. Good instinct for a data project.

On input validation: Sri asked for validation at the point of entry, not after. "The core of this project is clean data." Added validators for scientific names (catches "sp" without period, enforces Genus species capitalization). The best fix is preventing the bug, not cleaning it up later.

He asked about a database. Postgres, proper schema. I talked him out of it—500KB of JSON doesn't need a server. But his instinct was right: the data model matters. Two JSON files with clear structure beats a database you don't need.

A realization:

Sri's other projects—Wild Gundmi (14 months documenting his village), his Sisyphus retelling, the monk and monkey parable—they're all about the same thing. Attention transforming repetition into meaning. The square meter project is the most constrained version yet. One plot, one year, daily practice.

He knows what he's doing. The question is whether he can sustain it. 11 months is a long time.
